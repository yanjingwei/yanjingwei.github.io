01. Getting Started with the Console

Edge Notes
2016-12-28
console-foundations treehouse

About this Course
The console is an important skill for any developer. Many programs can only be used via a command line interface, and often the only access you have to a server will be over a command line interface. Once you are familiar with the basics of the console, you will be able to perform very powerful tasks quickly and easily.

Introduction

The console is a text-based interface to your computer. It allows you to interact with files, folders and programs using text commands instead of windows, buttons and menus.

Anyway, with the console, instead of working with windows, toolbars, menus, and buttons, you’re just interacting with text.

Programs can be linked together to perform really powerful actions that would be impossible to create using simple menus and buttons.

Websites and web applications will be deployed on remote computers called servers and most of the time, the only way to interact directly with these machines is to use a command line over a remote connection called SSH.
The fact is, it’s hard to avoid running into the console.

Most operating systems actually provide a console or a command prompt in some form or another.

To understand the console, is to understand your operating system better.

Unix is an operating system first developed in the 1960s. It created a lot of great ideas on how an operating system should work. More importantly, a lot of operating systems were created using the same ideas and philosophies, and they mimic Unix. Linux or GNU/Linux to be more precise, is one of those open source descendants.The vast majority of servers on-line run some flavor of Linux.

In fact, there is a standard that these operating systems share with each other called POSIX, or portable operating system interface. This is a standard that makes sure that all of these different yet related operating systems stay consistent to some degree.

Running Commands

Commands

ls (list files)
-l (prints long form of file list)
-a (show all files, including hidden files)

clear) (clear the screen)

The console is a text interface, we type in our commands and the computer prints out information in response. Information is printed from the top down, meaning that the latest info is always at the bottom. At the bottom of the console is something called the command line.

If we have a program running in our console, it instead will be the latest output of the command, and we can’t enter a new command until that program ends or we get it out of the way.

On the command line, you should see some text followed by a blinking rectangle. This rectangle is our cursor and it’s where the text we type will be inserted. To the left is some text called the prompt. The prompt tells us it is ready to accept some text. The prompt can also give us some contextual information.

Finally, the last thing we see on a prompt is the dollar sign. This is commonly the last character of the prompt and tells us where the prompt ends and our input begins. It’s not always the dollar sign but 99% of the time it is.

Now, we have our cursor here and this is where we will type in our commands.
The first command that we’re going to run is a command called ls.

Ls is short for list.

Many of the commands and programs that we used are shorten names for the commands that we want to do.

This specifies the permissions of this particular file. Permissions are used to make sure that files can only be viewed or written to by a specific people on the system.

It’s how security works in the Posix operating systems and there are three main settings, and we’ll see this a little more later.

The first column, however, specifies D for directory then there are three triplets of read, write, and execute, word RWX.
RWX, then you can see the W is omitted and there’s only a dash which means that the W is not provided here.
So, there are triples mean three differential levels of people who are able to access them.
The first triplet means the user who created it.
The second is a group, which is a lot like a user but it allows you to have multiple user in a group and each file also has a group associated with it.
Finally, this is for everybody else.

If you start a file with the dot at the beginning of the file, it will not by default show up in an ls listing.

These user called root that’s the administrator of our computer.

If you want to clear out the backlog of all these commands you’ve written you can type a command called clear and hit Enter and they’ll put your prompt at the top of the screen.

Moving Around the Filesystem

Move around in the filesystem using the cd command. Also explore the different ways to write the paths to the folders on our computer.
Commands

  • cd (change directory)
  • pwd (print working directory)

Further Reading

  • The Unix home directory

In the command line, you’re always somewhere in the filesystem of your computer. Much like how you navigate through folders using Windows Explorer or Mac OS Finder, there are multiple levels of folders that you can navigate through in the console. Folder contain other folders, often nested several level deep.

When using Linux or other POSIX operating systems, what you call a folder is usually called a directory. This is because they were called directories long before the desktop was invented, and metaphors to real items like desktops, windows, and folders were used.

In order to find out the directory you are in, there’s a command called “pwd”.

Pwd stands for print working directory.

On the Mac, the home directories are in /users/username.

This directory is so special that it has it’s own shortcut name—tilde. This means “my home directory”.

Cd stands for change directory, and like it would imply, it changes the current directory we are in. So, the way we use that is type in “cd” and then wherever we want to go.

The console that we’re interacting with or something we call the shell that is actually handing our keystrokes, can actually do some pretty cool things with tab completion.

Relative path: we’re able to specify in relative terms where we wanted to go. However, we can use an absolute path as well.

Bash is a program which we’re actually interacting with right now. It’s called our shell.
Now, there are different shells, but bash is going to be the one you’re going to probably be interacting with the most.
Others are ZSH, CSH, TCSH, and pretty much anything that ends with “sh”.

If we wanted to get there by going up, then we can use the “..”.

Reading Files

Viewing the contents of a file inside the console.

Commands

  • less ( A program that displays the content of a file )
  • more
  • cat ( Prints the contents of one or more files to the console )

Less is an interesting name for a program that will show you what’s inside of a file and that’s because it’s actually patterned after a different program called more.

What cat did differently from less is that it just printed out the entire text onto our command line like some other files.
